# Acts 2028 Sheepdog Society

## Overview
A weekly editorial newsletter for Christian men, anchored in Acts 20:28. Brief at `/Users/drewgodwin/Downloads/compass_artifact_wf-145eb503-8b48-455b-b02c-82c124aca57a_text_markdown.md`. Public site (Letter, Devotionals, Groups, Resources, Subscribe) plus a member community (chat, prayer, accountability, channels, reading plans) preserved from the original build. Admin-only auth via email + password (Credentials provider).

## Stack
- **Framework**: Next.js 16 (App Router, Turbopack) + TypeScript strict
- **UI**: shadcn/ui + Tailwind CSS v4 + Radix UI + Lucide icons
- **Brand**: Pasture & Iron palette + broadsheet-editorial system (2026-07-08 elevation): Fraunces variable (display/masthead, opsz/SOFT/WONK axes), Inter (folio/UI/body), Cormorant Garamond (pull-quotes/ember-band verse), Merriweather (reader serif), JetBrains Mono (section marks). Design truth: design-system/sheepdog-society/MASTER.md. Furniture utilities are Tailwind v4 @utility (cascade-layer safe).
- **Auth**: Auth.js v5 (NextAuth) Credentials provider â€” email + bcryptjs password, JWT sessions, allowlist via `ADMIN_EMAILS`. No adapter, no magic-link.
- **Database**: Neon Postgres in production (`DATABASE_URL` = pooled endpoint, host suffix `-pooler`; `DATABASE_URL_UNPOOLED` available for migrations). Wired into Vercel via the Marketplace integration so env vars sync automatically. Previous Supabase install was retired 2026-05-08.
- **ORM**: Drizzle ORM (`src/db/schema.ts`, 38 tables)
- **AI**: Vercel AI SDK (`ai` + `@ai-sdk/anthropic`) streaming via `claude-sonnet-4-5`. NEVER LangChain.
- **Email**: Resend (transactional + Broadcasts) + React Email templates (`src/emails/`)
- **Maps**: Mapbox GL + react-map-gl (Phase E geocoder/draggable-pin admin not yet built; existing `LocationMap` component covers reads)
- **Storage**: Vercel Blob (`@vercel/blob`)
- **Theme**: next-themes, defaultTheme="system" + enableSystem (2026-07-08); every public surface has true dark variants; AA gated by `npm run check:contrast` (scripts/check-contrast.mjs, both themes).

## Key Patterns
- Server Components by default; `"use client"` only for interactivity (editor, modals, state)
- Auth.js `auth()` available at `@/auth` (full Session) or `@/lib/auth-compat` (Clerk-shape `{ userId }` for the 76 legacy call sites still in transition)
- Admin gating: middleware lets public routes through; admin pages double-check `users.role === "admin"` server-side
- AI calls: server-only, all log to `ai_generations` with prompt/version/model/tokens. Banned-word list in `src/lib/ai/system-prompt.ts`. Programmatic banned-language gate at `src/lib/ai/banned.ts` (resources field notes + all letter-series generation). **Anthropic structured output rejects ALL size bounds** (array minItems/maxItems, string min/max, integer min/max â€” zod v4 `.int()` auto-emits the latter): every AI schema stays bounds-free; validate post-hoc with retry. Letter autopilot: `src/server/letters/autopilot.ts` (engine, `runAutopilot({dryRun})`), state in `letter_autopilot` (single row), series writes via `createSeriesWithLettersCore` (transactional, advisory lock 815551 shared with the wizard + single-letter create). Visibility email is load-bearing (failure reverts the block to drafts); covers best-effort after it. Resource field notes: AI-drafted + admin-approved via `src/lib/resources/generate-field-notes.ts`; only status `approved` renders publicly; admin hand-edits sanitized by `src/lib/resources/sanitize-field-notes.ts`. Bible verse text NEVER generated â€” use `{{VERSE: ref}}` placeholders
- Members & email (2026-07-22, #71): `members.subscribed` (default true) is the weekly-letter checkbox in `/admin/members`, mirrored best-effort into the Resend Audience (`src/lib/resend-audience.ts`) â€” the audience remains the broadcast target for the weekly letter. `/admin/announcements` sends branded shepherd@ blasts (all subscribed / leaders / men in groups) via `src/server/announcements.ts` + `src/emails/announcement.tsx`, with stateless HMAC unsubscribe links (`src/lib/announcements/helpers.ts`, AUTH_SECRET-keyed). Plant requests (`/join?path=start`) collect full group fields; Approve in `/admin/location-requests` geocodes (address â†’ city/state fallback) and auto-creates the group via `upsertGroupLocation` â€” idempotent via `location_requests.created_group_id`. Server-side Mapbox geocoding MUST send the site Referer (token is URL-restricted; handled in `src/lib/geocoding.ts`).
- Soft delete: `deletedAt` column + partial unique indexes (`.where(deleted_at IS NULL)`); 30-day cron purge
- Letter versions: every autosave writes a row to `letter_versions` for restore/diff
- Site text: curated editable copy (homepage + About ONLY). Registry in `src/lib/site-text/registry.ts` is the source of truth for keys/labels/defaults; `site_text` table stores overrides only (empty/whitespace = use default; blank-save deletes the row). Read via `getSiteTextMap()` (`src/lib/site-text/get.ts`, `unstable_cache` tag `site-text`); saves in `src/server/site-text-admin.ts` call `updateTag("site-text")` â€” NOT `revalidateTag`, which in Next 16 requires a profile arg and would serve stale copy after saves. Editor at `/admin/site-text`. Scripture is never a site_text key.
- Design Studio (`/admin/studio`, DS-1 + DS-2 + DS-3 + DS-4 shipped â€” COMPLETE, ALL 12 public pages governed (home/about/join/faq/contact/giving/what-to-expect/how-we-gather/events/letter/stories/resources) + an AI layer â€” see `docs/superpowers/specs/2026-07-11-admin-design-studio-design.md`): draft/publish site theme+layout with preview, side-by-side compare against live, and full undo. `site_studio` (single row, `draft`/`published` jsonb configs), `studio_versions` (snapshot history, bigint identity ordering), `site_text.draft_value` (draft overlay on the existing site-text table). Render-merge rule (`src/lib/studio/config.ts`) makes locked sections (Scripture, page heroes, live feeds/forms) permanently unhideable/unreorderable regardless of what's in the config â€” `src/lib/studio/sections.ts`'s `SECTION_REGISTRY` covers all 12 pages. Draft visibility is Next's `draftMode()`, checked strictly OUTSIDE every `unstable_cache` scope (never inside â€” that's the anti-poisoning rule); the theme `<style>` override renders ONLY from `src/app/(public)/layout.tsx`, never the root layout. Compare's "live" pane works via a middleware cookie-strip (`?studio=published` in `src/middleware.ts`) â€” not a loader-level check, since layouts can't read query params. Apply/Restore are Server Actions serialized by `pg_advisory_xact_lock(815552)` (NOT 815551, which belongs to letter series) with cache flush `updateTag("studio") + updateTag("site-text") + revalidatePath("/", "layout")`. 5 hand-designed themes (`src/lib/studio/themes-data.mjs`) cover 10 of 11 `--c-*` vars + all 20 shadcn semantic vars â€” `--c-iron` and the ember bands are theme-constant by design; the contrast gate (`check-contrast.mjs`) covers all 5 themes Ă— both modes. Ships disabled-by-default: empty config = today's site, unchanged. `/admin/studio`'s page selector (`PAGES` const in `page.tsx`) drives which page's sections/text/preview-iframe render; `saveDraftConfig` always takes and patches the WHOLE `StudioConfig` object (`{ ...config, pages: { ...config.pages, [selectedPage]: { sections } } }`) â€” never a partial object, or every other page's saved layout is lost. Contact is the one page split into a server `page.tsx` + a `src/components/public/contact-form.tsx` client component, since it was the only fully-client public page (needed the async Studio data fetch). Gotcha class hit 3 times during DS-2: a site-text default's WORDING can match the real page copy while its JSX STRUCTURE doesn't (e.g. a two-line `<br/>`+`<em>` headline collapsed into one key) â€” always check the actual surrounding markup shape, not just the rendered words, before wiring any headline/empty-state copy. AI layer (`src/server/studio-ai.ts`): `recommendForPage`/`assistField`/`describeChangeset`, all `requireAdmin`-gated, all logged to `ai_generations` (migration 0023 adds `studio_recommend`/`studio_assist`/`studio_describe` enum values â€” must be applied before the AI strip is used, auto-runs via `.github/workflows/apply-migrations.yml` on merge to main). AI never Applies â€” only calls the existing `saveDraftConfig`/`saveDraftText`. Changeset validation core (`src/lib/studio/changeset.ts`) enforces id existence, locked-section immunity, position bounds against the real `renderMerge`-materialized order, duplicate rejection, and a 20-item cap before anything reaches a draft; every AI string passes `findBannedLanguage`; every AI schema is bounds-free. `describeChangeset` reads the draft directly from the DB (`db.select().from(siteStudio)...normalize(row?.draft)`, matching `/admin/studio`'s own page.tsx) â€” NEVER `getStudioConfig()`, which only returns the real draft when `draftMode()`'s preview cookie happens to be on, not during normal editing; using it as the base silently reverts unstaged changes. The Design Studio arc (DS-1â€“DS-4) is complete; `discardDraft` now shares the same `pg_advisory_xact_lock(815552)` as `applyDraft`/`restoreVersion`.
- Resend Broadcasts: created in `publishLetter` server action; failures don't block the website publish
- Bible reader: public + stateless at `/bible` (no bookmarks/notes/highlights). Pure data/parsers in `src/lib/bible/books.ts` (Vitest), fetch in `chapter.ts`: ESV 24h cache â†’ WEB public-domain fallback with a visible notice.
